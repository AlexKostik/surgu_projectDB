# surgu_projectDB

Проект "surgu_projectDB" — учебная работа по проектированию базы данных для СТО (сервис технического обслуживания автомобилей).

Кратко:
- **Тема:** СТО — учёт клиентов, автомобилей, сотрудников, услуг и запчастей.
- **Цель:** разработать ER‑схему и реализовать структуру в PostgreSQL (файл с DDL — см. ниже).

Что включено:
- **ER‑схема:** визуальная диаграмма, описывающая сущности и связи (каталог: `ER/`).
- **SQL:** скрипт создания таблиц и ограничений — [SQL/projectDB.sql](SQL/projectDB.sql).

Схема проекта
- ER‑диаграмма: в папке `ER` находится файл с графическим изображением модели данных. Визуализация помогает понять связи между таблицами и правила удаления/целостности.

Список сущностей (из схемы и SQL):
- `clients` — клиенты сервиса
- `cars` — автомобили (ссылка на владельца в `clients`)
- `employees` — сотрудники сервиса
- `services` — перечень услуг
- `parts` — запчасти и их остатки на складе
- `orders`, `order_services`, `order_parts` — заказы и состав заказа (услуги + запчасти)

Как запустить локально
1. Убедитесь, что установлен PostgreSQL.
2. Создайте базу данных и примените скрипт:

```bash
createdb surgu_projectdb
psql -d surgu_projectdb -f SQL/projectDB.sql
```

3. После выполнения в базе появятся таблицы согласно ER‑схеме. Дальше можно наполнять тестовыми данными и писать запросы.

Файлы и структура
- `SQL/projectDB.sql` — скрипт создания таблиц и внешних ключей.
- `ER/` — папка с ER‑диаграммой (PNG/Draw.io).
- `README.md` — этот файл с описанием проекта.
